MyriadSwitcher

MyriadSwitcher is a python app to mine the wonderful crypto-currency known as Myriadcoin. Unlike most cryptos out there, Myriadcoin uses 5 different algorithms to mine the coin, namely SHA-256, Scrypt, Groestl, Skein and Qubit.

MyriadSwitcher fetches the web to determine which of the 4 GPU profitable algos to mine (Scrypt, Groestl, Skein or Qubit). You can aim for maximum coins per day, maximum coins per watt, or mine in a hybrid mode that's anything in between the former modes.

It also monitors your miners and automatically restarts them if they crash or freeze, and can generate an html output log to monitor your mining progress from anywhere.

 

Launch the application:

Double click the executable named MyriadSwitcherGUI.exe

 

Configuration

Main Config Panel

 

  1. Use the check boxes to select the algos you want to the switcher to pick from. (MANDATORY)
  2. Then select the script files to launch your miners for every enabled algo (MANDATORY)

  3. If you know your hash rate (in Mega-hashes per seconds) for each algo, input those those in the MH/s selectors. If  MyriadSwitcher knows your hash rate accurately it can make better decisions as to what mine or when to switch to a different algo.

    You'll also get more accurate predictions for expected coins per day and other stats displayed in the progress console that will be explained later in this document. If you don't know your hash rates or you're simply too lazy to set them, you can use the provided

    default values. The switcher will stil work, and will assume hash rates that match what Myriadcoin estimates for algo hash rate equivalencies at http://myriad.theblockexplorer.com/

  4.  Input your power consumption in the Watts selectors for each algo and your computer idle power consumption if you know them. As with hash rates, you can use defaults if you don't know them. It will not have an effect on how the app switches algos if you pick the Max Coins per Day mining mode. 

  5. Efficiency: We all know we don't really get all the coins we're supposed to based only on our hash rate. There's some rejected shares, blocks that go orphaned, pool fees and pool donations... By default this is set to 95% meaning that all your earnings are cut down by a 5% (so you get a 95% of the theoretical figure). 

 

Switching Modes Panel

 

Here you can pick your mining mode.

  • Maximize coins per day: self explanatory, the miner will simply pick the algorithm that will net you the most coins, depending on your hash rates and the algo's difficulties.
  • Maximize coins per watt: the switcher will priorize algos with lower power consumption like Groestl and Qubit. For example, if you have a system that draws 500W mining Groestl and 1000W mining Scrypt, Scrypt coins per day should be at least twice as many as Groestl's for the switcher to pick Scrypt over Groestl. In this example, Groestl has more weight for the switcher than Scrypt. 
  • Hybrid mode: it's anything in between (a) Maximize coins per day and (b) Maximize coins per watt. Use the slider to tweak how much you want it to be more like (a) or (b). 

Minimum Coins: If profitability gets too low for your taste, the switcher stops mining. Just set a minimum amount of coins per day below which you'd rather not have your system crunching hashes. If none of your active algos are currently able to generate more than this many coins per day, nothing is mined until difficulty goes down again.

Expected Values: this is just an informative section of the configurator.

  • Weight represents the priority of each algo for the switcher. If you are mining in Maximize coins per day mode, all weights will be equal simply because there's no preference for any algo, will just mine the most profitable, but in Maximize coins per watt mode, algos with lower power draw have more weight
  • Min. Coins: If you have set a minimum coin threshold and you are not using the Maximize coins per day mode, you can see here at which point the switcher will stop mining  for each algo.
  • Min C/W: Means Minimum Coins per Watt, same as before but in terms of Coins per Watt

 

Miscellaneous Panel

Timings

  • Normal Sleep time (minutes): This is the time in minutes that passes before the switcher checks again if there's a more profitable algo.
  • Sleep time after miner restarts (minutes): Same as before but right after switching to a new algo, or restarting the miner after spending some time bellow the Minimum Coins threshold. This is usually better set higher than the previous sleep time to make sure you spend at least this amount of time mining something and keep the switcher from swaping algos too quickly.
  • Switching Hysteresis (%): This is also to avoid frantic switching and is better explained with an example. If you set this to 10% and you're constantly mining say Scrypt at 1000 coins per day, another algo would need to be at least a 10% more profitable before it kicks in, so 1100 coins per day. If now for example Skein hits 1100 coins per day and takes over, Scrypt would need to be another 10% more profitable than Skein, so 1210 per day.
  • Min. hysteresis time (minutes): Once this many minutes have passed mining a certain algo, hiysteresis is switched off. Makes sense if you've been mining something for long enough to not worry about quick switching back and forth anymore.
  • Ramp up time (seconds): The time it takes your miners to restart and begin submiting shares. It's used only for statistical purposes, to factor in down time in the amount of coins per day expected as displayed in the progress console that will be described in a while.
  • HTTP timeout (seconds): The time the switcher waits for the interweb overlords to respond when querying for the MYR exchange rate and algo's difficulties.

Log Configuration:

      Pretty self explanatory. Enable or disable logging and pick the log destination folder. The log files are stored per session, and the naming follows the following pattern: [YEAR]-[MONTH]-[DAY]-[HOURS][MINUTES][SECONDS]. Example:2014-05-14-154545.html 

Exchange source

     This is to pick the source of the current MYR exchange rate vs Bitcoin. Currently available sources are Poloniex and Mintpal. If you want another one and it had an API, let me know and I'll add it. It agin just for statistical purposes anyway.

Miner Error Handling

     Select "Miner error monitoring" if you want MyriadSwitcher to restart your miners if they crash or freeze.

     You can also set the switcher to reboot your system completely after a number of consecutive errors, and specify if it should reboot after the miner crashes (the process is killed completely), freezes (the process halts and stops using CPU cycles) or both. 

     Setting "Reboot after 1 error" will  reboot the system as soon as a miner fails, with no attempt to restart it.

     After rebooting, the switcher will start automatically, wait a bit to give the system time to load, and start mining again  

 

Buttons

    Save: saves your current configuration.

    Cancel: discards the changes made to the configuration and reverts to the previously saved values.

    Defaults: Restores the config default values

    Run: Saves current changes made to the configuration panels and starts a mining session

    Stop: Terminates the mining session.

Note: You don't need to stop the mining session to apply changes, just reconfigure on the fly, hit Save and the new configuration will apply

 

Progress Console Output

As you start mining, the Console will output some data events about your current session progress in a tabular fashion, let's describe each column's meaning:

  • Time: The local time of the event.
  • Elapsed/Stint: The time in days hours-minutes-seconds since the session started, or the current coin stint started for summary lines
  • Algo: The algorithm being mined currently
  • Exch: The current exchange rate for Myriadcoin vs Bitcoin  in satoshis 
  • Prof 1Mh/s: Current expected profitability of the algo being mined in satoshis per 1MH/s per day.
  • My profit: Current expected profitability of the algo being mined in satoshis per day based on your own hash rate
  • Scrypt    %   /day: Total expected coins mined in Scrypt, it's percentage of the expected total coins mined for all 4 algos
  • Groestl   %   /day: Total expected coins mined in Groestl , it's percentage of the expected total coins mined for all 4 algos
  • Skein     %   /day: Total expected coins mined in Skein, it's percentage of the expected total coins mined for all 4 algos
  • Qubit     %   /day: Total expected coins mined in Qubit, it's percentage of the expected total coins mined for all 4 algos
  • Tot.Coins:  Total expected coins mined, including all 4 algos
  • Tot. $: Total expected satoshis, including all 4 algos, based on current exchange rate
  • /day: Expected coins per day  of the session/stint
  • Prof 1Mh/s: Total expected profitability of the session in satoshis per 1MH/s per day.
  • My profit: Total expected profitability of the session in satoshis per day based on your own hash rate
  • Watts: Average power consumption of the session or stint
  • C/W:  Average coins per watt of the session or stint

What I call a stint is simply the time spent on a given  algo, before the session ends, the switcher swaps to another one or you disable it and start mining something else. You can call it Sue if you want.

What's with the colorful green/cyan lines? These are summary stats for the stints and are displayed every time you switch algos. The values displayed are averages and totals just for that particuar... Sue 

And the red/magenta ones? Those mark a miner restart after it craps out, the switcher monitors their activity and restarts the miners if they crash or freeze.

 

Menu Items

File

  • Open: Open a new config file. By default the switcher uses myriadSwitcher.conf, but you can have any number of configs stored and use any of them as you see fit. The frame title includes the name of the current config file in use.
  • Save: Does the same as the Save button.
  • Save as: Saves your current configuration to a file with a name of your choice and sets it as the active config file
  • Exit: Closes the app and the miners. If you shut down the app clicking the "X" on the top right/left of the frame, the app closes but the miners keep running
